# KOSPI · KOSDAQ Monitor v4 적용 가이드

기존 `kospi-monitor` 리포에 **코스닥 + 테마/섹터 + 기준봉(+6%)** 을 더한 업데이트입니다.
`dart_scorer.py`, `tier_mapping.py`, `docs/`, `data/` 는 **건드리지 않습니다.**

## 1. 파일 교체 / 추가

| 파일 | 작업 |
|---|---|
| `kospi_check.py` | **교체** (코스피+코스닥 2시장, 테마/섹터, 기준봉) |
| `theme_fetcher.py` | **신규 추가** (네이버 테마/섹터 캐시 수집) |
| `.github/workflows/daily.yml` | **교체** (실행 타임아웃 20→40분, 400종목 대응) |

나머지 파일은 그대로 둡니다.

## 2. 무엇이 바뀌나

- **코스닥 추가**: 상단 `코스피 / 코스닥` 탭으로 전환. 각 시총 상위 **200**.
- **테마/섹터**: 종목명 옆 칩으로 표기 (예: SK하이닉스 `AI` `HBM` `반도체`).
  네이버에서 받아 `data/theme_cache.json` 에 저장하고 **7일마다 갱신**. 실패 종목은 비워둔 채 진행.
- **기준봉**: 전일 종가 대비 **등락률 +6% 이상**인 종목 행을 **붉은색**으로 강조하고, 별도 "🔥 기준봉" 표와 통계 카드도 추가.

## 3. 첫 실행은 PC에서 권장

DART 자동 점수와 테마 캐시를 처음 채우는 작업이라, 첫 회는 로컬에서 `run.bat`(또는 `python kospi_check.py`)으로 돌려 `fin_cache/`, `data/theme_cache.json` 을 데운 뒤 push 하시면, 이후 GitHub Actions는 캐시를 써서 빠르게 끝납니다.

## 4. 미리보기

`preview_index.html` 은 네트워크 없이 **샘플 데이터**로 렌더한 결과입니다(실데이터 아님).
실제 모양·기준봉 강조·테마 칩·시장 탭을 브라우저에서 먼저 확인하실 수 있습니다.

## 5. 참고

- 코스닥 종목은 수동 티어가 없어 대부분 `?`(미분류)로 표시됩니다. 필요하면 `tier_mapping.py`에 코스닥 코드 추가 가능.
- 테마 기준값/개수는 `theme_fetcher.py`의 `CACHE_TTL_DAYS`, `MAX_THEMES`에서, 기준봉 퍼센트는 `kospi_check.py`의 `ANCHOR_PCT`에서 조정.
- 네이버 JSON/HTML 구조가 바뀌어 테마가 안 채워지면 `theme_fetcher.py`의 셀렉터만 손보면 됩니다.

## 6. (v4.1) 실행 타임아웃 수정 — 예산제

첫 실행이 40분 타임아웃 난 원인: 캐시가 비어 400종목 테마(400회)+DART(최대 1,200회)를 한 번에 호출.

해결: **실행당 신규 호출 수 제한(예산제)** + 캐시를 리포에 커밋해 영구 저장.
- 워크플로 env: `THEME_MAX_NEW=120`, `DART_MAX_NEW=80` (코스피+코스닥 **합산** 기준, 실행당)
- 부족분은 다음 실행으로 이월 → 며칠에 걸쳐 캐시가 채워지고, 이후엔 캐시만 읽어 빠르게 끝남
- 이미 받은 종목은 매번 재사용(테마 14일, DART는 fin_cache 버전 일치 시)
- 커밋 대상에 `data/`(테마 캐시·스냅샷), `fin_cache/` 추가 → 캐시 유실 방지
- 타임아웃 40→30분

원하면 더 빨리 채우려고 `THEME_MAX_NEW`/`DART_MAX_NEW`를 키우거나, PC에서 한 번 풀로 돌려(예산 없이) 캐시를 채운 뒤 push 해도 됩니다.

## 7. (v4.2) 테마/섹터 숫자 표기 버그 수정

증상: 칩에 이름 대신 숫자(예: 삼성전자 `278`)가 표시됨.
원인: 네이버 모바일 API가 테마 자리에 숫자 코드를 반환 → 잘못 채택.

수정(**`theme_fetcher.py`만 교체**):
- 모바일 API 제거. 데스크톱 종목 페이지의 업종(`type=upjong`)·테마(`type=theme`) **링크 텍스트(한글)**만 사용.
- 강력 필터: 한글/영문자가 없는 값(숫자·기호만)은 절대 채택 안 함 → 숫자 칩 불가능.
- 캐시 스키마 버전 도입: 기존 숫자 캐시는 자동 무효화 후 재수집. 읽는 시점에도 필터링하므로 재수집 전에도 숫자는 화면에 안 나옴.

선택: 즉시 깨끗하게 다시 채우려면 `data/theme_cache.json`을 지우고 다시 실행하면 됩니다(안 지워도 점진적으로 교체됨).

참고: 데스크톱 페이지에 테마 링크가 없는 종목은 테마가 비고 업종만 표시될 수 있습니다(숫자가 나오던 문제와 무관, 정상). 테마가 전반적으로 비면 셀렉터(`type=upjong`/`type=theme`)만 조정하면 됩니다.

## 8. (v4.3) 테마/섹터 한글 깨짐 수정

증상: 칩이 깨진 한자/◆(예: "釩◆◆泥덕…")로 표시됨.
원인: 가져온 페이지의 인코딩 오판독 + 일부 잘못된 요소 채택.

수정(**`theme_fetcher.py`만 교체**):
- 인코딩 자동 보정: 페이지 바이트를 utf-8/cp949로 디코딩해 한글이 가장 많은 쪽 채택.
- 필터 강화: **한자·◆·숫자·기호만인 값은 전부 거부** → 어떤 경우에도 깨진 값/숫자가 표시되지 않음(통과 조건: 한글 또는 영문 포함 2~20자).
- 소스: 모바일 JSON(구조적, 우선) → 데스크톱(보조). 캐시 스키마 버전업으로 기존 깨진 캐시 자동 무효화 + 읽는 시점 재필터.

권장: `data/theme_cache.json`을 한 번 지우고 다시 실행하면 즉시 깨끗하게 다시 채워집니다.

> 참고: 네이버가 제 환경에서 차단돼 라이브 검증은 못 했습니다. **깨진 값/숫자가 사라지는 것은 보장**되며, 실제 업종/테마가 채워지는지는 다시 실행한 화면으로 확인이 필요합니다. 만약 업종까지 비어 보이면, 네이버 DOM 대신 KRX(pykrx) 업종 분류로 섹터를 받는 방식으로 전환하면 100% 정확하게 채울 수 있습니다.

## 10. (v4.5) 생성 시각 KST 표기 수정

증상: "생성" 시각이 한국시간과 9시간 어긋남(예: 화면 12:39 = 실제 KST 21:39).
원인: GitHub Actions 러너가 UTC라 `datetime.now()`가 UTC를 표기.

수정(**`kospi_check.py`만 교체**):
- KST(UTC+9) 헬퍼 도입. 헤더 생성 시각을 `… KST 생성`으로 한국시간 표기.
- 기준일 판정(`get_effective_date`)도 KST 기준으로 통일(오전 9시 이전엔 전 거래일).

참고(스케줄): 워크플로는 평일 **16:00·17:00·18:00·19:30 KST** 4회 실행(무료 크론 지연·누락 대비 다중 등록)이라, 화면의 "생성 시각"은 그날 마지막으로 성공한 실행 시각으로 표시됩니다. GitHub 무료 크론은 정시보다 수십 분 지연될 수 있어 18:00이 아니라 18:2x 등으로 찍힐 수 있습니다. 18:00 한 번으로만 보이게 하려면 cron을 `0 9 * * 1-5` 하나만 남기면 됩니다(단, 신뢰성은 떨어짐).
